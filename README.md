## Overview

Staqtapp-TDS is a high-performance, Python-centric storage engine built
around a deterministic native core. The architecture cleanly separates
the native storage engine from Python orchestration, browser operations,
telemetry, future native intelligence modules and future C-based VM search
& extract driver language/registry/builder for renowned trace ranking
evolutions across .tds/.tdd/.tdr files.

## Highlights

-   Deterministic native storage engine
-   Python-first development interface
-   Browser Operations Console
-   Real-time telemetry
-   Native diagnostics
-   Modular internationalization
-   CPU-first architecture
-   Optional intelligence modules

## Version 2.8.1

This release focuses on hardening the Browser Operations Console before
the implementation of the Native Spiral Rank Engine.

### Security Improvements

-   CSRF protection for administrative POST operations
-   Origin/Referer validation
-   Safe DOM rendering replacing unsafe dynamic HTML paths
-   Hardened request parsing
-   Improved browser language settings validation

The native storage engine and storage format are unchanged.

## Previous Release (v2.8.0)

-   Browser localization framework
-   English, Japanese, Spanish, Portuguese, German, French and Italian
-   Modular language packs

## Design Philosophy

The storage engine remains deterministic and authoritative.

Browser telemetry, diagnostics, and future intelligence modules observe
the engine---they never become part of the storage hot path.
